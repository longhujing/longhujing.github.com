买卖股票最佳时机汇总

买卖股票的最佳时机 Ⅰ

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

示例 1：

输入：[7,1,5,3,6,4]
输出：5
解释：在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。
注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。

示例 2：

输入：prices = [7,6,4,3,1]
输出：0
解释：在这种情况下, 没有交易完成, 所以最大利润为 0。

简单来说，就是买一次卖一次，求最大的利益。这种情况很容易象到就是保证买入代价最小，卖出价格尽可能高。

假设i ~ j范围内买入价格最低为k，则该范围内买入的最大利润为Math.max(price[i] - k, price[i + 1] - k, ...)

在该题中，在第i天的时候只会有两种状态：持有股票或者不持有股票。假设dp[i][0]表示第i天不持有股票的最大收益，dp[i][1]表示第i天持有股票的最大收益，那么可以很轻松得到如下转换式子：

  • 对于不持有股票来说，有两种情况

    1. 昨天就不持有股票，所以今天是延续昨天的状态，即dp[i][0] = dp[i - 1][0]
    2. 昨天持有股票，但是今天卖出去了，即dp[i - 1][0] = dp[i - 1][1] + price[i]

    因此最终得出的表达式为

    dp[i][0] = Max(dp[i - 1][0], dp[i - 1][1] + price[i])

  • 对于持有股票来说，也有两种情况

    1. 昨天就持有股票，而只能进行一次交易，所以今天是昨天状态的延续，即dp[i][1] = dp[i - 1][1]
    2. 昨天没有持有股票，是今天买入的，由于只能进行一次交易，因此今天是第一次买入，则今天的收益必然是-price[i]，即dp[i][1] = -price[i]

    因此最终得出的表达式为

    dp[i][1] = Max(dp[i - 1][1], -price[i])

买卖股票的最佳时机 Ⅱ

给定一个数组，它的第 i 个元素是一支给定股票第 i 天的价格。

设计一个算法来计算你所能获取的最大利润。你可以尽可能地完成更多的交易（多次买卖一支股票）。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

示例 1:

输入: [7,1,5,3,6,4]
输出: 7
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 3 天（股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。
随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候卖出, 这笔交易所能获得利润 = 6-3 = 3 。

示例 2:

输入: [1,2,3,4,5]
输出: 4
解释: 在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。
注意你不能在第 1 天和第 2 天接连购买股票，之后再将它们卖出。
因为这样属于同时参与了多笔交易，你必须在再次购买前出售掉之前的股票。

示例 3:

输入: [7,6,4,3,1]
输出: 0
解释: 在这种情况下, 没有交易完成, 所以最大利润为 0。

题目有个前提是下一次交易必须是在上一次交易完成以后才能进行。用我们正常人的思维来看的话，只要有得赚，就一定要入手，什么情况有的赚？涨的时候有的赚，所以这边的核心就是买涨不买跌，只要有的赚就必须买。

在该题中，在第i天的时候只会有两种状态：持有股票或者不持有股票。假设dp[i][0]表示第i天不持有股票的最大收益，dp[i][1]表示第i天持有股票的最大收益，那么可以很轻松得到如下转换式子：

  • 对于不持有股票来说，可能有两种情况

    1. 昨天也不持有股票，那么今天就是延续昨天的状态，即dp[i][0] = dp[i - 1][0]
    2. 昨天持有股票，但是今天卖出去了，即dp[i][0] = dp[i - 1][1] + price[i]

    因此最终得出表达式为dp[i][0] = Max(dp[i - 1][0], dp[i - 1][1] + price[i])

  • 对于持有股票来说，也有两种情况

    • 昨天也持有股票，那么今天肯定是没有购入股票的，因此今天的状态延续昨天的状态，即dp[i][1] = dp[i - 1][1]
    • 昨天没有持有股票，今天购入了股票，即dp[i][1] = dp[i - 1][0] - price[i]

    因此最终得出表达式为dp[i][1] = Max(dp[i - 1][1], dp[i - 1][0] - price[i])

买卖股票的最佳时机 Ⅲ

给定一个数组，它的第 i 个元素是一支给定的股票在第 i 天的价格。

设计一个算法来计算你所能获取的最大利润。你最多可以完成 两笔 交易。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

示例 1:

输入：prices = [3,3,5,0,0,3,1,4]
输出：6
解释：在第 4 天（股票价格 = 0）的时候买入，在第 6 天（股票价格 = 3）的时候卖出，这笔交易所能获得利润 = 3-0 = 3 。
随后，在第 7 天（股票价格 = 1）的时候买入，在第 8 天 （股票价格 = 4）的时候卖出，这笔交易所能获得利润 = 4-1 = 3 。

示例 2：

输入：prices = [1,2,3,4,5]
输出：4

解释：在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。
注意你不能在第 1 天和第 2 天接连购买股票，之后再将它们卖出。
因为这样属于同时参与了多笔交易，你必须在再次购买前出售掉之前的股票。

示例 3：

输入：prices = [7,6,4,3,1]
输出：0
解释：在这个情况下, 没有交易完成, 所以最大利润为 0。
示例 4：

输入：prices = [1]
输出：0

第三种场景和第一种场景很类似，只不过这边最多只能买入两次，但是思路是一样的，就是多了几种状态而已。即第一次买入股票-未持有0、第一次买入股票-已持有1、第二次买入股票-未持有2、第二次买入股票-已持有3这4种状态。

  • 第一次买入股票–未持有股票，有以下几种情况

    1. 昨天就没有持有股票，今天是昨天状态的延续，即dp[i][0] = dp[i - 1][0]
    2. 昨天持有第一次买入的股票，今天卖掉了，即dp[i][0] = dp[i - 1][1] + price[i]

    因此最终得出的表达式为

    dp[i][0] = Max(dp[i - 1][0], dp[i - 1][1] + price[i])

  • 第一次买入股票–已持有股票，有以下几种情况

    1. 昨天就持有第一次买入的股票，今天是昨天状态的延续，即dp[i][1] = dp[i - 1][1]
    2. 昨天没有持有股票，今天是第一次买入股票，即dp[i][1] = -price[i]

    因此最终得出的表达式为

    dp[i - 1][1] = Max(dp[i - 1][1], -price[i])

由于股票交易必须建立在上一次交易完成之后，因此第二次的交易状态转换与第一次稍有不同，具体体现如下

  • 第二次买入股票–未持有股票

    1. 昨天未持有股票，今天是昨天状态的延续，而此时的昨天状态应该是卖出第一次入手的股票的状态，即dp[i - 1][2] = dp[i - 1][0]
    2. 昨天持有第二次买入的股票，今天卖掉了，即dp[i - 1][2] = dp[i - 1][3] + price[i]

    因此最终得出的表达式为

    dp[i - 1][2] = Max(dp[i - 1][0], dp[i - 1][3] + price[i])

  • 第二次买入股票–已持有股票

    1. 昨天已经持有第二次买入的股票，今天是昨天状态的延续，即dp[i][3] = dp[i - 1][3]
    2. 昨天未持有股票，今天是第二次买入股票，因此今天的最大收益应该是第一次卖出股票的最大收益-买入今天股票的费用，即dp[i][3] = dp[i - 1][0] - price[i]

    因此最终的出的表达式为

    dp[i - 1][3] = Max(dp[i - 1][3], dp[i - 1][0] - price[i])

除了转换表达式稍微复杂一点，还有一些需要注意的地方

  1. 第一次、第二次买卖股票的顺序不能相反，要按照正常的顺序
  2. 初始的时候，即i = 0的时候，第二次交易绝对不可能进行，所以此时第二次交易的初始值为Integer.MIN_VALUE

买卖股票的最佳时机 Ⅳ

给定一个整数数组 prices ，它的第 i 个元素 prices[i] 是一支给定的股票在第 i 天的价格。

设计一个算法来计算你所能获取的最大利润。你最多可以完成 k 笔交易。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

示例 1：

输入：k = 2, prices = [2,4,1]
输出：2
解释：在第 1 天 (股票价格 = 2) 的时候买入，在第 2 天 (股票价格 = 4) 的时候卖出，这笔交易所能获得利润 = 4-2 = 2 。

示例 2：

输入：k = 2, prices = [3,2,6,5,0,3]
输出：7
解释：在第 2 天 (股票价格 = 2) 的时候买入，在第 3 天 (股票价格 = 6) 的时候卖出, 这笔交易所能获得利润 = 6-2 = 4 。
随后，在第 5 天 (股票价格 = 0) 的时候买入，在第 6 天 (股票价格 = 3) 的时候卖出, 这笔交易所能获得利润 = 3-0 = 3 。

和买卖股票的最佳时机 Ⅲ的思路是一样的，唯一的区别就是前者限制死了只能买两次而已，其实再分析一下上一道题，不难发现即使调整为最多买3次，其实整个的流程都是一样的，即先买后卖，第一次买入股票的收益可能是-price[i]，第二次及之后买入股票的最大收益可能是前一次卖出股票扣除当前买入股票的费用

最佳买卖股票时机含冷冻期

给定一个整数数组，其中第 i 个元素代表了第 i 天的股票价格 。

设计一个算法计算出最大利润。在满足以下约束条件下，你可以尽可能地完成更多的交易（多次买卖一支股票）:

你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。
卖出股票后，你无法在第二天买入股票 (即冷冻期为 1 天)。
示例:

输入: [1,2,3,0,2]
输出: 3
解释: 对应的交易状态为: [买入, 卖出, 冷冻期, 买入, 卖出]

这道题状态转移过程画图来看是这样的

这边有个冷冻期，冷冻期的含义是什么？就是上一次持有股票到这一次持有股票至少要隔一天。因此我们需要一个中间状态，即卖出股票的状态。上图的转换状态就能够表示这一次持有股票和上一次持有股票至少隔了一个状态（一天）。

假设dp[i][0]表示第i天不持有股票，dp[i][1]表示第i天持有股票，dp[i][2]表示第i天为卖出股票的状态。

  • 对于持有股票来说，可能有两种情况

    1. 昨天就持有股票，今天是昨天状态的延续，因此dp[i][1] = dp[i - 1][1]
    2. 昨天未持有股票，今天才买入的，因此dp[i][1] = dp[i - 1][0] - prices[i]

    因此总结为: dp[i][1] = Max(dp[i - 1][1], dp[i - 1][0] - prices[i])

  • 对于卖出股票来说，从图中的转换过程来说，只能是从持有到当前的情况，因此dp[i][2] = dp[i - 1][1] + prices[i]

  • 对于未持有股票来说，可能有两种情况

    1. 昨天就未持有股票，今天是昨天状态的延续，因此dp[i][0] = dp[i - 1][0]
    2. 昨天是卖出了股票，所以今天就没有股票，因此dp[i - 1][0] = dp[i - 1][2]

    因此总结为: dp[i][0] = Max(dp[i - 1][0], dp[i - 1][2])

买卖股票的最佳时机含手续费

给定一个整数数组 prices，其中第 i 个元素代表了第 i 天的股票价格 ；非负整数 fee 代表了交易股票的手续费用。

你可以无限次地完成交易，但是你每笔交易都需要付手续费。如果你已经购买了一个股票，在卖出它之前你就不能再继续购买股票了。

返回获得利润的最大值。

注意：这里的一笔交易指买入持有并卖出股票的整个过程，每笔交易你只需要为支付一次手续费。

示例 1:

输入: prices = [1, 3, 2, 8, 4, 9], fee = 2
输出: 8
解释: 能够达到的最大利润:
在此处买入 prices[0] = 1
在此处卖出 prices[3] = 8
在此处买入 prices[4] = 4
在此处卖出 prices[5] = 9
总利润: ((8 - 1) - 2) + ((9 - 4) - 2) = 8.

还是和之前的思路一样，这边有手续费，但由于手续费是一样的，因此可以算在买入成本，也可以算在卖出成本，这边简单起见就算在买入成本里面。

还是假设dp[i][0]为第i天未持有股票的最大收益，dp[i][1]为第i天持有股票的最大收益，那么

  • 对于未持有股票来说，可能有两种情况

    1. 昨天就未持有股票，今天是昨天状态的延续，因此dp[i][0] = dp[i - 1][0]
    2. 昨天持有股票，今天卖出了，因此dp[i][0] = dp[i - 1][1] + prices[i]

    因此总结为: dp[i][0] = Max(dp[i - 1][0], dp[i - 1][1] + prices[i])

  • 对于持有股票来说，可能有两种情况

    1. 昨天就持有股票，今天是昨天状态的延续，因此dp[i][1] = dp[i - 1][1]
    2. 昨天未持有股票，今天才买入，由于我们把手续费算在买入成本里面了，因此这边买入的过程还需要加如手续费，即dp[i][1] = dp[i - 1][0] - prices[i] - fee

    因此总结为: dp[i][1] = Max(dp[i - 1][1], dp[i - 1][0] - prices[i] - fee)

其实这样分析，即便每一笔交易的手续费不一样，甚至每一笔的买入和卖出的手续费不一样，还是能够按照这种思路解决。
